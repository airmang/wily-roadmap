# Wily Roadmap Orchestrator Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make Wily explicitly own roadmap orchestration while delegating phase implementation planning to external planners.

**Architecture:** Keep `.wily/roadmap.yaml` as the project-level plan and make phase folders external-planner friendly. `scripts/wily.py` should create and pass a context bundle, but it should not generate detailed implementation plans.

**Tech Stack:** Python standard library, Markdown files, `unittest`.

---

### Task 1: External Planner Tests
